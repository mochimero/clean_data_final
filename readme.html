READ ME FIRST!

This repo contains 2 R scripts and a codebook.
  • run_analysis.R is the main script. You must set the working directory at the original path of the data folders. This is, once you UNZIP the file, go to the UCI HAR Dataset FOLDER and set this one as the WORKING DIRECTORY. Either use the setwd() command or do it through the GUI in RStudio or R
  • getting_summarised_tiby_data.R is a script that must run AFTER the run_analysis.R script. It will create a smaller data set where you could see the mean values of the MEASUREMENTS for each activity and each subject.
  • codebook describes the final variable names, their units etc.

    Going from raw data to tidy data

    1. First of all, download the zip file from the following URL: https://d396qusza40orc.cloudfront.net/getdata%2Fprojectfiles%2FUCI%20HAR%20Dataset.zip
    2. Unzip the file and set the working directory in R (or RStudio) as the UCI HAR Dataset folder
    3. Source the run_analysis.R script. This script is the main one and it does the following steps:
      1. It reads the files activity_labels.txt and features.txt. It collects from features the names of some of the variables (the mean and the std of the tBodyAccXYZ, tBodyGyroXYZ and tGravityAccXYZ)
      2. Later it goes to each subfolder (train and test subfolders) and read the subject files (that is, a file with numbers from 1 to 30 which identify the subjects of the experiment), the feature data (it collects only the mean and std described in the previous step), and the activities vector (a file with numbers from 1 to 6 for each measurement).
      3. The activities file is later changed from integer labesl to string labels using the convention described in the activity_labels.txt file. It set the names to lowercase and remove all characters like undersocre, dots, etc
      4. The subject column remains untouched, it is always formed by integers from 1 to 30.
      5. The scripts goes to the Inertial Signals folder, where the actual measurements are stored.
      6. All the files are read and combined into a data frame with 1152 columns (128 columns per file x 9 files)
      7. It does the same steps for the second folder
      8. After both folders are scanned and files are read, the final data frame with 1172 variables is built (it is called full_df). Also a data table is built with the data.table package.
      9. The names for the final data frame are set
      10. As a fnal step, all the R objects are removed, leaving only the full_df data frame and its table version
    4. Once the script ends, you will have a data frame called full_df. There is no filtering by mean or std parameters.
    5. After this script finishes, you can run the getting_summarised_tidy_data.R script in order to get a data frame that shows the mean of each measurement for each activity and subject.
    6. This data frame has 180 rows and 1154 columns. The first column is the activity, the second is the subject label and the rest are the mean values
